 4  """Bio.SearchIO support for InterProScan output formats. 
 5   
 6  This module adds support for parsing InterProScan XML output. 
 7  The InterProScan is available as a command line program or on 
 8  EMBL-EBI's web page. 
 9  Bio.SearchIO.InterproscanIO was tested on the following version: 
10   
11  - versions: 5.26-65.0 (interproscan-model-2.1.xsd) 
12   
13  More information about InterProScan are available through these links: 
14  - Publication: https://www.ncbi.nlm.nih.gov/pmc/articles/PMC3998142/ 
15  - Web interface: https://www.ebi.ac.uk/interpro/search/sequence-search 
16  - Documentation: https://github.com/ebi-pf-team/interproscan/wiki 
17   
18   
19  Supported format 
20  ================ 
21   
22  Bio.SearchIO.InterproscanIO supports the following format: 
23   
24  - XML   - 'interproscan-xml' - parsing 
25   
26   
27  interproscan-xml 
28  ================ 
29   
30  The interproscan-xml parser follows the InterProScan XML described here: 
31  https://github.com/ebi-pf-team/interproscan/wiki/OutputFormats 
32   
33  +----------------+--------------------+--------------------------------------+ 
34  | Object         | Attribute          | XML Element                          | 
35  +================+====================+======================================+ 
36  | QueryResult    | target             | 'InterPro'                           | 
37  |                +--------------------+--------------------------------------+ 
38  |                | program            | 'InterProScan'                       | 
39  |                +--------------------+--------------------------------------+ 
40  |                | version            | protein-matches.interproscan-version | 
41  +----------------+--------------------+--------------------------------------+ 
42  | Hit            | accession          | signature.name                       | 
43  |                +--------------------+--------------------------------------+ 
44  |                | id                 | signature.ac                         | 
45  |                +--------------------+--------------------------------------+ 
46  |                | description        | signature.desc                       | 
47  |                +--------------------+--------------------------------------+ 
48  |                | dbxrefs            | IPR:entry.ac                         | 
49  |                |                    | go-xref.id                           | 
50  |                |                    | pathway-xref.db:pathway-xref.id      | 
51  |                +--------------------+--------------------------------------+ 
52  |                | attributes         |                                      | 
53  |                | ['Target']         | *-match / *-location                 | 
54  |                | ['Target version'] | signature-library-release.library    | 
55  |                | ['Hit type']       | signature-library-release.version    | 
56  +----------------+--------------------+--------------------------------------+ 
57  | HSP            | bitscore           | *-location.score                     | 
58  |                +--------------------+--------------------------------------+ 
59  |                | evalue             | *-location.evalue                    | 
60  +----------------+--------------------+--------------------------------------+ 
61  | HSPFragment    | query_start        | *-location.start                     | 
62  | (also via      +--------------------+--------------------------------------+ 
63  | HSP)           | query_end          | *-location.end                       | 
64  |                +--------------------+--------------------------------------+ 
65  |                | hit_start          | *-location.hmm-start                 | 
66  |                +--------------------+--------------------------------------+ 
67  |                | hit_end            | *-location.hmm-end                   | 
68  |                +--------------------+--------------------------------------+ 
69  |                | query              | sequence                             | 
70  +----------------+--------------------+--------------------------------------+ 
71   
72  InterProScan XML files may contain a match with multiple locations or multiple 
73  matches to the same protein with a single location. In both cases, the match is 
74  uniquely stored as a HIT object and the locations as HSP objects. 
75   
76  HSP.*start == *start - 1 (Since every start position is 0-based in Biopython) 
77  HSP.aln_span ==  query-end - query-start 
78   
79  The types of matches or locations (eg. hmmer3-match, hmmer3-location, 
80  coils-match, panther-location) are stored in hit.attributes['Hit type']. 
81  For instance, for every 'phobious-match', there will be a 'phobious-location'. 
82  Therefore, Hit.type will store the string excluding '-match' or '-location' 
83  ('phobious', in this example).
