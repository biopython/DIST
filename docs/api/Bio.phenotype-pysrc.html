  9  Input 
 10  ===== 
 11  The main function is Bio.phenotype.parse(...) which takes an input file, 
 12  and format string.  This returns an iterator giving PlateRecord objects: 
 13   
 14      >>> from Bio import phenotype 
 15      >>> for record in phenotype.parse("phenotype/Plates.csv", "pm-csv"): 
 16      ...     print("%s %i" % (record.id, len(record))) 
 17      ... 
 18      PM01 96 
 19      PM09 96 
 20   
 21  Note that the parse() function will invoke the relevant parser for the 
 22  format with its default settings.  You may want more control, in which case 
 23  you need to create a format specific sequence iterator directly. 
 24   
 25  Input - Single Records 
 26  ====================== 
 27  If you expect your file to contain one-and-only-one record, then we provide 
 28  the following 'helper' function which will return a single PlateRecord, or 
 29  raise an exception if there are no records or more than one record: 
 30   
 31      >>> from Bio import phenotype 
 32      >>> record = phenotype.read("phenotype/Plate.json", "pm-json") 
 33      >>> print("%s %i" % (record.id, len(record))) 
 34      PM01 96 
 35   
 36  This style is useful when you expect a single record only (and would 
 37  consider multiple records an error).  For example, when dealing with PM 
 38  JSON files saved by the opm library. 
 39   
 40  However, if you just want the first record from a file containing multiple 
 41  record, use the next() function on the iterator (or under Python 2, the 
 42  iterator's next() method): 
 43   
 44      >>> from Bio import phenotype 
 45      >>> record = next(phenotype.parse("phenotype/Plates.csv", "pm-csv")) 
 46      >>> print("%s %i" % (record.id, len(record))) 
 47      PM01 96 
 48   
 49  The above code will work as long as the file contains at least one record. 
 50  Note that if there is more than one record, the remaining records will be 
 51  silently ignored. 
 52   
 53  Output 
 54  ====== 
 55  Use the function Bio.phenotype.write(...), which takes a complete set of 
 56  PlateRecord objects (either as a list, or an iterator), an output file handle 
 57  (or in recent versions of Biopython an output filename as a string) and of 
 58  course the file format:: 
 59   
 60          from Bio import phenotype 
 61          records = ... 
 62          phenotype.write(records, "example.json", "pm-json") 
 63   
 64  Or, using a handle:: 
 65   
 66          from Bio import phenotype 
 67          records = ... 
 68          with open("example.json", "w") as handle: 
 69             phenotype.write(records, handle, "pm-json") 
 70   
 71  You are expected to call this function once (with all your records) and if 
 72  using a handle, make sure you close it to flush the data to the hard disk. 
 73   
 74   
 75  File Formats 
 76  ============ 
 77  When specifying the file format, use lowercase strings. 
 78   
 79   - pm-json - Phenotype Microarray plates in JSON format. 
 80   - pm-csv  - Phenotype Microarray plates in CSV format, which is the 
 81               machine vendor format 
 82   
 83  Note that while Bio.phenotype can read the above file formats, it can only 
 84  write in JSON format.
