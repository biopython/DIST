  6  """Bio.SearchIO support for HMMER output formats. 
  7   
  8  This module adds support for parsing HMMER outputs. HMMER is a 
  9  suite of programs implementing the profile hidden Markov models to find 
 10  similarity across protein sequences. 
 11   
 12  Bio.SearchIO.HmmerIO was tested on the following HMMER versions and flavors: 
 13   
 14      - HMMER3 flavors: hmmscan, hmmsearch, phmmer 
 15      - HMMER2 flavors: hmmpfam, hmmsearch 
 16   
 17  More information on HMMER are available through these links: 
 18    - Web page: http://hmmer.janelia.org/ 
 19    - User guide: ftp://selab.janelia.org/pub/software/hmmer3/3.0/Userguide.pdf 
 20   
 21   
 22  Supported formats 
 23  ================= 
 24   
 25  Bio.SearchIO.HmmerIO supports the following HMMER output formats: 
 26   
 27      - Plain text, v3.0   - 'hmmer3-text'    - parsing, indexing 
 28      - Table, v3.0        - 'hmmer3-tab'     - parsing, indexing, writing 
 29      - Domain table, v3.0 - 'hmmer3-domtab'* - parsing, indexing, writing 
 30      - Plain text, v2.x   - 'hmmer2-text'    - parsing, indexing 
 31   
 32  * For the domain table output, due to the way HMMER outputs the sequence 
 33    coordinates, you have to specify what HMMER flavor produced the output as the 
 34    file format. So instead of using 'hmmer3-domtab', you have to use either 
 35    'hmmscan3-domtab', 'hmmsearch3-domtab', or 'phmmer3-domtab' as the file format 
 36    name. 
 37   
 38  Note that for all output formats, HMMER uses its own convention of input and 
 39  output coordinates. It does not use the term 'hit' or 'query', instead it 
 40  uses 'hmm' or 'ali'. For example, 'hmmfrom' is the start coordinate of the HMM 
 41  sequence while 'alifrom' is the start coordinate of the protein sequence. 
 42   
 43  HmmerIO is aware of this different naming scheme and will adjust them 
 44  accordingly to fit SearchIO's object model. If HmmerIO sees that the output file 
 45  to parse was written by hmmsearch or phmmer, all 'hmm' coordinates will be the 
 46  hit coordinates and 'ali' coordinates will be the query coordinates. Conversely, 
 47  if the HMMER flavor is hmmscan, 'hmm' will be query and 'ali' will be hit. 
 48   
 49  This is why the 'hmmer3-domtab' format has to be specified with the source HMMER 
 50  flavor. The parsers need to know which is the hit and which is the query. 
 51  'hmmer3-text' has its source program information present in the file, while 
 52  'hmmer3-tab' does not output any coordinates. That's why both of these formats 
 53  do not need direct flavor specification like 'hmmer3-domtab'. 
 54   
 55  Also note that when using the domain table format writers, it will use HMMER's 
 56  naming convention ('hmm' and 'ali') so the files you write will be similar to 
 57  files written by a real HMMER program. 
 58   
 59   
 60  hmmer2-text and hmmer3-text 
 61  =========================== 
 62   
 63  The parser for HMMER 3.0 plain text output can parse output files with alignment 
 64  blocks (default) or without (with the '--noali' flag). If the alignment blocks 
 65  are present, you can also parse files with variable alignment width (using the 
 66  '--notextw' or '--textw' flag).
